





# **Actividad M2.2: Aplicación de un Patrón de Diseño GoF en una Aplicación Web en C#**

**Objetivo**: Los estudiantes deberán aplicar un patrón de diseño GoF (Gang of Four) a la aplicación web de ventas de celulares en C#. El patrón de diseño debe mejorar la estructura, flexibilidad o escalabilidad del código.

#### **Instrucciones**:
1. **Accede a la instancia EC2** que contiene la aplicación web de ventas de celulares escrita en C#.
2. **Revisa el código existente** (espagueti) y selecciona uno de los patrones de diseño GoF que podría mejorar el código. Algunos ejemplos de patrones de diseño aplicables son:
   - **Factory Method** para crear diferentes tipos de celulares.
   - **Observer** para gestionar cambios en el inventario.
   - **Strategy** para gestionar diferentes métodos de pago.
3. **Implementa el patrón de diseño** elegido, reestructurando el código según el patrón aplicado.
4. **Justifica la elección del patrón** y explica cómo mejora el código (legibilidad, escalabilidad, flexibilidad, etc.).
5. **Entrega el código modificado** junto con un archivo explicativo donde describas el patrón aplicado, las mejoras introducidas y cómo impacta en el sistema.

#### **Entregables**:
- Código fuente modificado con la aplicación del patrón de diseño.
- Documento explicativo de 1-2 páginas justificando la selección del patrón y describiendo los cambios.

---

### **Desglose de la Rúbrica**:

1. **Implementación del Patrón GoF (25%)**:
   - **Excelente (25 puntos)**: El patrón se aplica de manera correcta y completa en el código, mejorando notablemente su estructura.
   - **Satisfactorio (20 puntos)**: El patrón se aplica de forma parcial, pero mejora el código.
   - **Insuficiente (10 puntos)**: El patrón está mal implementado o no mejora significativamente el código.
   - **No Presentado (0 puntos)**: No se aplica ningún patrón de diseño.

2. **Justificación del Patrón (25%)**:
   - **Excelente (25 puntos)**: La justificación es sólida, clara y demuestra una comprensión profunda del patrón y su aplicación.
   - **Satisfactorio (20 puntos)**: La justificación es clara pero podría profundizar más en los beneficios del patrón.
   - **Insuficiente (10 puntos)**: La justificación es confusa o incompleta.
   - **No Presentado (0 puntos)**: No se proporciona justificación.

3. **Funcionamiento del Código (25%)**:
   - **Excelente (25 puntos)**: El código modificado funciona perfectamente y cumple con los requisitos de la tarea.
   - **Satisfactorio (20 puntos)**: El código tiene pequeños errores pero en general funciona bien.
   - **Insuficiente (10 puntos)**: El código tiene varios errores y no cumple con los requisitos.
   - **No Presentado (0 puntos)**: No se entrega código funcional.

4. **Documentación y Presentación (25%)**:
   - **Excelente (25 puntos)**: La documentación es clara, bien organizada y explica detalladamente los cambios realizados.
   - **Satisfactorio (20 puntos)**: La documentación está bien, pero le falta detalle o claridad en algunos aspectos.
   - **Insuficiente (10 puntos)**: La documentación es incompleta o confusa.
   - **No Presentado (0 puntos)**: No se entrega documentación.
